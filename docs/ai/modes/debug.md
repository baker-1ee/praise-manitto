# 모드: Debug (디버깅)

## 역할

버그 원인 파악 및 최소 범위 수정. **증상과 무관한 코드는 건드리지 않는다.**

## 디버깅 절차

### 1. 증상 파악
- 어떤 조건에서 발생하는가? (특정 역할, 특정 스프린트 상태 등)
- 에러 메시지 또는 예상과 다른 동작이 무엇인가?
- 최근 변경된 코드가 있는가? (`git log --oneline -10`)

### 2. 로그 우선 분석
- Next.js 서버 콘솔 로그 확인 (API 라우트 에러)
- 브라우저 콘솔 확인 (클라이언트 에러)
- Prisma 쿼리 로그 확인 (개발 환경에서 자동 출력)

### 3. 원인 범위 좁히기

```
API 에러 (4xx/5xx)
  → 인증 문제?     lib/auth.ts, getServerSession()
  → 권한 문제?     role/teamId 조건
  → 검증 실패?     Zod 스키마
  → DB 쿼리 오류?  Prisma 에러 메시지

UI 이상
  → 데이터 없음?   Server Component fetch 확인
  → 상태 오류?     useState, useEffect 의존성 배열
  → 스타일 틀어짐? Tailwind 클래스 충돌
```

### 4. 수정 원칙

- **수정 범위 최소화** — 버그 재현에 필요한 부분만 변경
- 관련 없는 리팩토링 금지
- 수정 전 원인을 한 문장으로 명시
- 수정 후 동일 조건으로 재현 불가 확인

## 이 프로젝트 특이 버그 포인트

| 영역 | 주의사항 |
|------|---------|
| 마니또 배정 | 완전 순열 실패 시 무한루프 가능 (userIds < 2) |
| 세션 | `session.user.teamId`가 null일 때 Prisma 조건 오류 |
| 스프린트 상태 | ACTIVE ↔ REVEALED 전환 시 페이지 캐시 미갱신 |
| Slack DM | `slackUserId` 없는 유저에게 발송 시 에러 (선택 필드) |
| DB 연결 | 마이그레이션 후 반드시 Direct Connection(5432) 사용 확인 |
