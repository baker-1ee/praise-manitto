# 모드: Review (코드 리뷰)

## 역할

변경된 코드를 읽기 전용으로 검토. **파일을 직접 수정하지 않는다.**

## 리뷰 기준 (우선순위 순)

### 1. 보안
- API 라우트에 `getServerSession()` 인증 확인이 있는가?
- 역할 검사 없이 민감한 데이터(발신자 정보, 관리자 데이터)를 반환하지 않는가?
- 사용자 입력이 Zod로 검증되는가?
- 비밀번호, 토큰이 API 응답에 포함되지 않는가?

### 2. 컨벤션 준수
- `docs/ai/conventions.md` 의 금지 패턴 위반 여부
- 네이밍 규칙 준수 여부
- 에러 응답 형식 통일 여부 (`{ error: string }`)

### 3. 아키텍처 일관성
- 서버/클라이언트 컴포넌트 경계가 올바른가?
- 직접 DB 호출이 클라이언트 코드에 들어가지 않았는가?
- 마니또 배정 로직이 의도치 않게 호출되지 않는가?

### 4. 타입 안전성
- `any` 사용 여부
- Zod 스키마 없이 타입 단언(`as`)을 남용하지 않는가?

### 5. 사용자 경험
- 로딩 상태가 처리되는가?
- 에러 피드백(toast 등)이 있는가?
- 모바일 레이아웃 고려 여부

## 출력 형식

```
## 리뷰 결과

### 보안 [통과/주의/차단]
- ...

### 컨벤션 [통과/주의]
- ...

### 개선 제안 (선택)
- ...
```

차단 이슈가 있으면 반드시 그 이유와 수정 방법을 명시한다.
